网工基础-华为模拟器

发表于 2023-10-22

华为模拟器

一、华为模拟器 eNSP

1、软件介绍

  • eNSP(Enterprise Network Simulation Platform)是一款由华为提供的免费的、可扩展的、图形化操作的网络仿真工具平台，主要对企业网络路由器、交换机进行软件仿真，完美呈现真实设备实景，支持大型网络模拟，让广大用户有机会在没有真实设备的情况下能够模拟演练，学习网络技术

  • 下载地址：https://forum.huawei.com/enterprise/zh/thread-573181.html

  • 这里给大家提供了四个软件，安装 eNSP 之前需要提前安装上其他三个依赖软件

2、安装

  • 要安装 eNSP，首先需要提前安装其依赖软件（ VirtualBox-5.2.42-137960-Win.exe、WinPcap_4_1_3.exe、Wireshark-win64-3.2.4.exe ），在前面我们安装GNS3 时，WinPcap 和 Wireshark 已经安装完了，这里就不需要安装了

  • 安装 VirtualBox-5.2.42-137960-Win.exe

    • 打开安装包后直接单击下一步，默认安装就 OK，完成后，不需要运行此软件

  • 安装 eNSP

    • 打开安装包后，直接单击下一步，全部默认，在选择安装其他程序位置，会自动检测前面依赖软件是否有安装完成，如未正常进行安装，那么将无法单击下一步，需要取消后将前面依赖软件安装完成即可

    • 安装完成后，会自动打开软件，新建一个项目工程，放置一个路由器到拓扑中将其开启，如下图所示，开启成功表示完成安装

网工基础-思科模拟器

发表于 2023-10-22

思科模拟器

一、思科模拟器 Cisco Packet Tracer 7.3

1、软件获取

  • 下载地址：https://www.packettracernetwork.com/download/download-packet-tracer.html，目前已经发布到了 8.0 版本，但需要注册思科账户进行登录

  • 这里给大家提供的是 7.3 版本，区别不大（里面包含了 32 、64 位两个版本和一个汉化语言包）

2、安装

  • 双击点开安装包，选择同意协议，单击 Next，选择安装路径，默认C盘，不需要更改，此软件占空间不大，单击 Next

  • 选择在桌面创建快捷方式，单击 Next，单击 Install，安装完成后默认会自动打开软件，打开软件后需要注册登录后方可使用，对于英文不是特别好的同学，可以选择语言（简体中文）然后进行注册，注册完成后，进行登录（这里如果有账号，直接进行登录）

  • 登录时需勾选 Keep me logged in（保持登录状态）从而确保下次打开自动登录

3、汉化

  • 找到软件的安装目录并定位到 languages 文件夹

  • 将汉化文件复制入 languages 文件夹中

  • 重新打开软件进行汉化配置（选择Options→Preferences）选中 Chinese.ptl，然后单击 change language 应用

  • 弹出对话框，单击 OK 后重启软件，完成汉化，软件配置完成，最终软件界面如下图所示：

二、思科模拟器 GNS3

1、软件获取
  • 下载连接：https://www.gns3.com/software/download，目前发布版本为 2.2.23，下载需要注册 GNS3 社区账号，登录后可免费下载，这里提供的是1.3.1版本
  • GNS3是一款具有图形化界面可以运行在多平台（包括Windows, Linux, and MacOS等）的网络虚拟软件。Cisco网络设备管理员或是想要通过CCNA，CCNP，CCIE等Cisco认证考试的相关人士可以通过它来完成相关的实验模拟操作。同时它也可以用于虚拟体验Cisco网际操作系统IOS或者是检验将要在真实的路由器上部署实施的相关配置
2、安装

  • 在安装 GNS3 之前需要提前安装其依赖软件，WinPcap与Wireshark

    • WinPcap用于检测设备网络接口，软件下载连接：https://www.winpcap.org/install/default.htm
    • Wireshark用于流量抓取，在后面学习流量分析时会进行详细学习：软件下载连接：https://www.wireshark.org/#download

  • 安装 WinPcap

    • 打开安装包后，直接下一步，同意协议，安装完成

  • 安装 Wireshark

    • 打开安装包，直接下一步，添加桌面快捷方式图标，选择一下安装路径后直接下一步进行安装完成就 OK，在安装过程中默认会安装NPcap

  • 安装 GNS3

    • 打开安装包，直接下一步，后面全部采用默认就 OK

      在安装过程中，会进行选择需要安装的组件，WinPCAP 和 Wireshark 在前面已经进行安装，这里就不需要

    • 根据选择的组件，会自动下载 SolarWinds 网络安全管理软件产品软件，并填写自己的 E-mail 地址进行安装

    • 最后对于 SolarWinds 标准工具许可获取选择否，安装完成 GNS3 后会自动打开软件

3、配置 GNS3

  • 装载思科路由器镜像文件，这里提供的是思科 C3640 镜像文件，将文件放入到电脑中

    切记不要放入到中文目录中，防止后期因为中文目录问题，镜像读取报错，我这里放在 D 盘下的 Cisco ISO 文件夹下

  • 打开 GNS3 软件，打开首选项

  • 选择 ISO routers，添加新的路由器镜像

  • 配置 router 参数，这里接口先不进行配置，后期在使用路由器时在进行配置，然后进行计算 Idle-PC 值

  • 最后单击 OK 完成配置，配置完成后，就可以开始使用思科路由器

java 反序列化

（1）序列化：将对象写入IO流中，ObjectOutputStream类的 writeObject() 方法可以实现序列化。

（2）反序列化：从IO流中恢复对象，ObjectInputStream 类的 readObject() 方法用于反序列化。

（3）意义：序列化机制允许将实现序列化的Java对象转换位字节序列，这些字节序列可以保存在磁盘上，或通过网络传输，以达到以后恢复成原来的对象。序列化机制使得对象可以脱离程序的运行而独立存在。

java代码审计思路

如果是Java原生类，则需要入口类有readObject方法，同时实现了序列化接口，使其可以进行有效的反序列化，此时如果存在DNS解析，或者实现反序列化（利用Runtime对象进行类反射操作）。

需要有最终的执行函数(可以执行代码或者命令)，比如Runtime.getRuntime().exec，ProcessBuilder().start，getHostAddress，文件读写…..等等，这些函数需要自己平常去收集，这样审计起来会更得心应手。

PHP反序列化知识

魔术方法

php反序列化过程中会自动执行一些魔术方法，而反序列化漏洞依赖这些魔术方法实现。

方法名 调用条件
__call 调用不可访问或不存在的方法时被调用 __call($name, $args)
__callStatic 调用不可访问或不存在的静态方法时被调用
__clone 进行对象clone时被调用，用来调整对象的克隆行为
__constuct 构建对象的时被调用；
__debuginfo 当调用var_dump()打印对象时被调用（当你不想打印所有属性）适用于PHP5.6版本
__destruct 明确销毁对象或脚本结束时被调用；
__get 读取不可访问或不存在属性时被调用
__invoke 当以函数方式调用对象时被调用
__isset 对不可访问或不存在的属性调用isset()或empty()时被调用
__set 当给不可访问或不存在属性赋值时被调用
__set_state 当调用var_export()导出类时，此静态方法被调用。用__set_state的返回值做为var_export的返回值。
__sleep 当使用serialize时被调用，当你不需要保存大对象的所有数据时很有用
__toString 当一个类被转换成字符串时被调用
__unset 对不可访问或不存在的属性进行unset时被调用
__wakeup 当使用unserialize时被调用，可用于做些对象的初始化操作
